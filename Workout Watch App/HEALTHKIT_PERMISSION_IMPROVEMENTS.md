# HealthKit権限とワークアウト起動の改善

## 実装した改善点

### 1. 権限管理の明確化 ✅

**変更前の問題:**
- `isAuthorized` が private Bool で、UIから状態が見えなかった
- 権限がない状態でもワークアウトを開始しようとして、タイムアウトが発生していた

**変更後の改善:**
```swift
@Published var authorizationStatus: AuthorizationStatus = .notDetermined

enum AuthorizationStatus {
    case notDetermined  // 未確定
    case requesting     // リクエスト中
    case authorized     // 許可済み
    case denied         // 拒否された
}
```

- UIで権限状態を監視可能
- ユーザーに現在の状態を明確に表示できる
- 権限がない場合、ワークアウトボタンを無効化

### 2. 権限リクエストのタイミング改善 ✅

**変更前:**
- アプリ起動時（init）にバックグラウンドで権限リクエスト
- ユーザーが気づかずにスキップする可能性

**変更後:**
- ワークアウト選択画面の表示時（onAppear）に権限リクエスト
- ユーザーが明示的に「権限を許可」ボタンをタップできる
- 権限が拒否された場合、専用のUIで再リクエストを促す

### 3. UIでの権限状態の可視化 ✅

**WorkoutTypeSelectionView の改善:**

1. **権限未確定の場合:**
   - 「HealthKit権限を許可」ボタンを表示
   - 画面表示時に自動的に権限リクエスト

2. **権限リクエスト中:**
   - ProgressView と「権限を確認中...」メッセージを表示

3. **権限拒否の場合:**
   - ⚠️ 「HealthKit権限が必要です」という警告
   - 「権限を許可」ボタンで再リクエスト可能
   - 説明テキストでなぜ権限が必要か説明

4. **権限許可済み:**
   - 通常のワークアウト選択リストを表示
   - すべてのワークアウトタイプが選択可能

### 4. タイムアウトの短縮 ✅

**変更前:**
```swift
let timeoutDuration: Duration = .seconds(10)  // 実機で10秒待機
```

**変更後:**
```swift
let timeoutDuration: Duration = .seconds(3)   // 実機で3秒に短縮
```

- ユーザー体験の改善：「準備中」の時間を大幅に短縮
- タイムアウト後も処理を続行（HealthKitはバックグラウンドで準備を続ける）
- 「データ収集を準備中...」という一時的なメッセージを5秒間表示し、自動消去

### 5. ワークアウト開始時の権限確認強化 ✅

**変更前:**
```swift
// 権限確認が曖昧で、確認後も3秒待機
if !isAuthorizedValue {
    await requestAuthorization()
    try? await Task.sleep(for: .seconds(3))  // 無駄な待機
}
```

**変更後:**
```swift
// 最優先で権限確認、拒否された場合は即座にエラー表示
let currentAuthStatus = await MainActor.run { self.authorizationStatus }

if currentAuthStatus != .authorized {
    await requestAuthorization()
    
    let newAuthStatus = await MainActor.run { self.authorizationStatus }
    if newAuthStatus == .denied {
        // 即座にエラーを返す（無駄な待機なし）
        await MainActor.run {
            self.errorMessage = "HealthKit権限が必要です。設定アプリから権限を付与してください。"
        }
        return
    }
}
```

- 権限がない場合、すぐにユーザーに通知
- 無駄な待機時間を削除
- より明確なエラーメッセージ

## 使用方法

### ユーザーの操作フロー

1. **アプリ起動**
   - WorkoutTypeSelectionView が表示される

2. **権限リクエスト（初回のみ）**
   - 自動的に権限リクエストダイアログが表示される
   - または「HealthKit権限を許可」ボタンをタップ

3. **権限を許可**
   - システムダイアログで「許可」をタップ
   - ワークアウト選択リストが有効になる

4. **ワークアウト開始**
   - 好みのワークアウトタイプをタップ
   - 速やかにワークアウトが開始される（3秒以内）

### 権限が拒否された場合

1. UIに警告メッセージが表示される
2. 「権限を許可」ボタンで再リクエスト可能
3. それでもだめな場合、設定アプリで手動で許可する必要がある

## テスト推奨事項

### 実機テスト

1. **初回起動テスト:**
   - アプリをアンインストール
   - 再インストール
   - 権限リクエストが表示されることを確認
   - 「許可」をタップしてワークアウトが開始できることを確認

2. **権限拒否テスト:**
   - 権限リクエストで「許可しない」をタップ
   - UIに警告が表示されることを確認
   - 「権限を許可」ボタンで再リクエストできることを確認

3. **ワークアウト開始速度テスト:**
   - 権限許可後、ワークアウトをタップ
   - 3秒以内にワークアウトビューに遷移することを確認
   - タイムアウトエラーが発生しないことを確認

4. **設定アプリでの権限変更テスト:**
   - 設定 > ヘルスケア > データアクセスとデバイス > [あなたのアプリ]
   - 権限をオフにする
   - アプリに戻って警告が表示されることを確認
   - 権限をオンに戻してワークアウトが開始できることを確認

## 期待される改善効果

1. **ユーザー体験の向上:**
   - 権限状態が明確に可視化される
   - タイムアウト時間が1/3に短縮（10秒→3秒）
   - エラー時の対処方法が明確

2. **トラブルシューティングの簡素化:**
   - 権限がない理由が明確
   - UIで権限リクエストを再試行可能
   - ログで権限状態を追跡可能

3. **信頼性の向上:**
   - 権限がない状態でワークアウトを開始しようとすることを防止
   - タイムアウト後も処理を続行（HealthKitのバックグラウンド処理を待つ）

## 注意事項

- HealthKitは**読み取り権限の状態を隠す**ため、完全な確認はできません
- **書き込み権限**（ワークアウトの保存）は確認可能です
- 実機では、HealthKitのデータ収集に数秒かかることがあります（GPSロック等）
- シミュレータでは制限された動作となります
